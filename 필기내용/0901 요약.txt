수업 필기 관련
- 윈도우 세팅 / 다이렉트엑스 세팅 / 화면출력이 어디서 어떻게 이루어지는지 알아두어야 함

Lost Device란
-  전체 -> 다른화면 -> 다시 전체화면으로 돌아가지 않음 => lost device
- 엑스 사용시에는 내가 복구해야 함 => 모니터 2개여야 고치기 수월함
---------------------------------------------------
- 어떠한 상황에서 하드웨어에 도달할 수 있는 수단이 망가져 있는 상황
=> 하나하나 확인해서 수정해 줘야 함(Update 맥락)
- Device Lost 처리 -> 실제 게임내에서도 자주 만나게 됨(Update 맥락인듯)		
- Device 상태 확인(Update마다) -> Device 상태에 문제 존재할 경우
- Device와 그 외 Device를 통해 생성된 모든 리소스를 복구함
=> 구조를 잘 알아두어야 함!

여태까지 한 것
- 다이렉트엑스를 이용한 베이스 작업
=> 작게나마 게임 엔진 만든 것!(윈도우용)

Sprite
- 일반적으로 게임에서 애니메이션을 사용하기 위해서 사용
- 일정영역으로 분리되어 있는 사진
(4장의 이미지 메모리 == 1장의 스프라이트 이므로 메모리 절약 효과가 있음)
+) Atlas : 공간 활용을 위해서 사진을 모아둔 것
+) 반드시 기억 -> 스프라이트, 알파채널(투명값), 텍스처(이미지)

- 너비와 높이가 중요함!(정확한 수치)
-> 하나 결정되면 나머지 스프라이트도 같은 비율을 따라야 함

+) 기능이 언급되지 않는다는 것 -> 캡슐화가 이루어 지는 것
ex) 프로그래머가 다이렉트엑스를 언급하지 않게 됨

- 있다고 가정하고 작성 -> 선언 필요 -> 존재 x 인것을 확인하고 생성함
